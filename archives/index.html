Linux环境下建立SSHFS挂靠的方法

Linux环境建立SSHFS共享的方法

客户端安装sshfs及fuse

  • CentOS环境安装命令：sudo yum install sshfs fuse
  • Ubuntu环境安装命令：sudo apt-get install sshfs fuse

    手动安装sshfs-fuse的方法：

  • tar -xzvf sshfs-fuse-2.4.tar.gz
  • cd sshfs-fuse-2.4
  • ./configure
  • make -j
  • sudo make install

配置ssh自动登陆

  • 在客户端使用普通用户生成密钥对,执行命令：ssh-keygen -d
  • 通过客户端，将公钥上传至服务端,
  • 执行命令：ssh-copy-id -i ~/.ssh/id_dsa.pub 服务器IP

建立sshfs共享

  • ssh username@服务器IP:/服务器中要共享的文件目录/ /共享到本地的文件目录/

java SSH BUG 出错及解决办法

java SSH项目 BUG出错及解决办法

1.问题：

Exception in thread "main" java.lang.NoClassDefFoundError:org/dom4j/DocumentException
at HibernateTest.main(HibernateTest.java:14)

解决方法：

添加dom4j.jar(解析hibernate.cfg.xml文件)

2．问题：

Exception in thread "main" java.lang.NoClassDefFoundError: org/apache/commons/logging/LogFactory
at org.hibernate.cfg.Configuration.<clinit>(Configuration.java:116)
at HibernateTest.main(HibernateTest.java:14)

解决方法

添加commons-logging.jar(记录解析过程)

3．问题：

Exception in thread "main" java.lang.NoClassDefFoundError: org/apache/commons/collections/SequencedHashMap

解决方法：

添加commons-collections.jar(在解析映射文件时，需要把所有持久化对象的定义放在一个HashMap中)

4．问题：

Exception in thread "main" java.lang.NoClassDefFoundError: net/sf/ehcache/CacheException

解决方法：

添加ehcache.jar(高速缓存，提高存取速度)

5．问题：

net.sf.ehcache.config.Configurator configure
警告: No configuration found. Configuring ehcache from ehcache-failsafe.xml found in the classpath: jar:file:/F:/MyJava/Hibernate/lib/ehcache-1.1.jar!/ehcache-failsafe.xml
Exception in thread "main" java.lang.NoClassDefFoundError

解决方法：

把ehcache-1.1.jar解压，把其中的ehcache-failsafe.xml改成ehcache.xml。

6.问题：

Caused by: java.lang.ClassNotFoundException: net.sf.cglib.transform.impl.InterceptFieldEnabled

解决方法：

添加cglib-full-2.0.02.jar

7.问题：

Caused by: java.lang.NoSuchMethodError: net.sf.cglib.proxy.Enhancer.setInterceptDuringConstruction(Z)V

解决方法：

添加cglib-nodep-2.1_3.jar

8.问题:

Exception in thread "main" java.lang.NoClassDefFoundError: javax/transaction/Synchronization

结局方法:

添加jta.jar(事务处理)

9.问题:

Exception in thread "main" java.lang.NoClassDefFoundError: antlr/ANTLRException

解决方法:

添加antlr-2.7.5h3.jar

10问题:

Caused by: java.sql.SQLException: The statement (1) has no open cursor.
at com.mysql.jdbc.MysqlIO.checkErrorPacket(MysqlIO.java:2901)
at com.mysql.jdbc.MysqlIO.checkErrorPacket(MysqlIO.java:773)

解决方法:

 <!-- <property name="jdbc.fetch_size">50 </property>  --> 把上面这个属性按上面这样注释掉就可以了.
